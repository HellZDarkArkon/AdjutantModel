#lang en_US

# en_US.prosody — per-word stress pattern and pitch accent override dictionary
#
# PURPOSE
#   Allows explicit lexical stress patterns and pitch accent assignments to be
#   associated with individual words, overriding the algorithmic outputs of
#   FootParser (stress placement) and PitchAccentModel (accent assignment).
#   Useful for:
#     - Function words that are always unstressed/deaccented in connected speech
#     - Words with irregular lexical stress FootParser cannot derive correctly
#     - Contextual prosodic highlighting (emphasis, contrastive stress, register)
#
# -----------------------------------------------------------------------
# FORMAT  (word_prosody section)
# -----------------------------------------------------------------------
#
#   word - S1 - S2 - ... - ACCENT
#
#   Token delimiter  :  ' - '  (space-hyphen-space)
#   Left token       :  lowercase word (matched case-insensitively)
#   Si tokens        :  one per syllable, left-to-right
#                         P = PRIMARY    → StressLevel::PRIMARY
#                         S = SECONDARY  → StressLevel::SECONDARY
#                         U = UNSTRESSED → StressLevel::UNSTRESSED
#   Final ACCENT token: pitch accent applied to the PRIMARY syllable
#                         H_STAR        → PitchAccentType::H_STAR
#                                         High peak; declarative medial default
#                         L_STAR        → PitchAccentType::L_STAR
#                                         Low valley; deaccented/secondary material
#                         L_PLUS_H_STAR → PitchAccentType::L_PLUS_H_STAR
#                                         Rising accent; phrase-initial, interrogative
#                         H_PLUS_L_STAR → PitchAccentType::H_PLUS_L_STAR
#                                         Falling-peak; phrase-final declarative
#                         DOWNSTEP_H    → PitchAccentType::DOWNSTEP_H
#                                         Downstepped high; follows prior H* in phrase
#                         NONE          → PitchAccentType::NONE
#                                         No pitch accent; unstressed function words
#                         AUTO          → Delegate to PitchAccentModel
#                                         Uses phrase-position + boundary-type rules:
#                                           tPhrase < 0.15             → L_PLUS_H_STAR
#                                           tPhrase ≥ 0.80, DECLARATIVE → H_PLUS_L_STAR
#                                           tPhrase ≥ 0.80, INTERROGATIVE → L_PLUS_H_STAR
#                                           preceded by H_STAR/DOWNSTEP_H → DOWNSTEP_H
#                                           otherwise                  → H_STAR
#
# CONTEXT-SENSITIVE VARIANTS  (phrase boundary type overrides)
#
#   word[DECLARATIVE]   - S1 - ... - ACCENT
#   word[INTERROGATIVE] - S1 - ... - ACCENT
#   word[CONTINUATION]  - S1 - ... - ACCENT
#   word[EXCLAMATORY]   - S1 - ... - ACCENT
#
#   Append a PhraseType label in square brackets to the word token.
#   Context variants are matched first; the plain entry is the fallback.
#   PhraseType is determined by the terminal punctuation of the phrase that
#   contains the word (see PhraseContour.h: PhraseType enum).
#
# SYLLABLE COUNT VALIDATION
#   The number of Si tokens must match the syllable count produced by
#   SyllableBuilder for the phoneme sequence in en_US.dict.  Mismatches
#   are silently ignored (entry treated as absent) and a debug warning
#   should be emitted when support for this file is implemented.
#
# -----------------------------------------------------------------------
# PIPELINE INTERACTION
# -----------------------------------------------------------------------
#
#   1. FootParser::Parse() → ProsodicWord  (algorithmic stress)
#      ↓ OVERRIDE: replace each syllable's StressLevel with Si from this file
#
#   2. PhraseContour::Multiplier() → phrase-level F0 base  (unchanged)
#
#   3. PitchAccentModel::Apply()   → per-syllable accent coarticulation
#      ↓ OVERRIDE: if ACCENT ≠ AUTO, inject the specified PitchAccentType
#                  directly into the primary syllable before coarticulation
#                  propagates to neighbouring accented syllables
#
#   4. IntonationModel::Compute()  → per-phoneme F0/amp/dur params  (unchanged)
#      Note: overriding stress level here also changes F0/amp/dur scales
#      via primaryF0Scale / secondaryF0Scale / unstressedF0Scale.
#
# -----------------------------------------------------------------------
# MODIFICATION GUIDE  (for "in solitude" prosodic shaping)
# -----------------------------------------------------------------------
#
#  To make "solitude" sound more meditative / receding:
#    Lower the primary F0 scale in IntonationModel params, or change
#    the declarative accent to H_PLUS_L_STAR here — the falling peak
#    drops pitch sharply after the stressed "sol", giving a sense of
#    resignation or quiet.
#
#  To make "solitude" sound upward-reaching / hopeful:
#    Change accent to L_PLUS_H_STAR — pitch rises INTO the stressed
#    syllable rather than falling from it, giving a searching quality.
#
#  To neutralise "solitude" (robotic, flat):
#    Change accent to L_STAR and all stress tokens to U — PitchAccentModel
#    assigns NONE, IntonationModel uses unstressedF0Scale throughout.
#
#  To make "in" carry contrastive stress ("in solitude, not together"):
#    Change the "in" entry to: in - P - H_STAR
#    FootParser will have already assigned a trochaic foot; this overrides
#    the stress level so IntonationModel lifts its F0/amp.
#
# -----------------------------------------------------------------------
#word_prosody

# -----------------------------------------------------------------
# "in solitude" — the target phrase
# -----------------------------------------------------------------

# "in"  /ɪn/  — 1 syllable
#   Preposition; fully unstressed in connected speech.
#   Receives no pitch accent and is metrically invisible to the phrase contour.
#   IntonationModel will apply unstressedF0Scale (0.88 × baseF0 ≈ 194 Hz).
in - U - NONE

# "solitude"  /ˈsɒl.ɪ.tuːd/  — 3 syllables: sol · i · tude
#   Phoneme sequence (en_US.dict): S - AA - L - IH - T - UW - D
#   Syllable 1  "sol"   → PRIMARY   (head of foot 1, trochee head)
#   Syllable 2  "i"     → UNSTRESSED (foot 1 weak position; reduced /ɪ/)
#   Syllable 3  "tude"  → SECONDARY  (head of degenerate foot 2; /tuːd/)
#
#   AUTO delegates to PitchAccentModel which will choose:
#     phrase-initial  → L_PLUS_H_STAR on "sol"  (rising onset)
#     phrase-medial   → H_STAR on "sol"          (high peak)
#     phrase-final declarative → H_PLUS_L_STAR   (falling peak, closed cadence)
#     phrase-final interrogative → L_PLUS_H_STAR  (open cadence)
solitude - P - U - S - AUTO

# Explicit phrase-type overrides for "solitude".
# Uncomment and edit to lock the accent regardless of phrase position.

# Declarative (sentence ends with '.' or ';'):
#   H_STAR — steady high peak on "sol"; good for mid-sentence occurrence.
#   H_PLUS_L_STAR — falling peak; use when "solitude" closes the sentence.
#   Change this line to select the accent quality you prefer:
solitude[DECLARATIVE]   - P - U - S - H_PLUS_L_STAR

# Interrogative (sentence ends with '?'):
#   Rising accent on "sol" keeps the phrase sounding open / questioning.
solitude[INTERROGATIVE] - P - U - S - L_PLUS_H_STAR

# Continuation (phrase ends with ',' or ':'):
#   Mild rise sustains forward motion without closing the utterance.
solitude[CONTINUATION]  - P - U - S - L_PLUS_H_STAR

# Exclamatory (sentence ends with '!'):
#   High peak with full amplitude — "sol" is loud and prominent.
solitude[EXCLAMATORY]   - P - U - S - H_STAR
